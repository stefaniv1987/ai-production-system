# Observability

Observability is required to monitor the behavior, performance, and quality of the AI system in production.

## Metrics

| Metric | Description |
|---|---|
| Latency | Time required to answer a user question |
| Throughput | Number of requests processed per second |
| Token Usage | Number of tokens consumed by LLM calls |
| Error Rate | Percentage of failed requests |
| Retrieval Accuracy | Quality of retrieved documents |
| Hallucination Rate | Percentage of unsupported or incorrect responses |

## Tools

- LangSmith for tracing chains and agent workflows.
- Arize AI for model quality monitoring and drift detection.

## Purpose

Observability helps detect regressions, optimize costs, debug issues, and improve the reliability of the AI pipeline.
